Installation

Unzip the download package. All the files of the website are in the 'Booom vX.X' folder. You can simply copy the contents of this folder to your web server or localhost. Once installed, visit "/index.html" to view the website.

HTML Structure

Booom! is built with the Bootstrap 3 framework - http://www.getbootstrap.com - that means you have access to all the great features, css & js goodies has to offer. To learn more about Bootstrap's grid system, please refer to http://getbootstrap.com/css/#grid

There is one single html document, where all the HTML for the site is present - this file is called index.html. The different sections of the website are clearly separated in commented sections, in the HTML and CSS files, to help you find your way easily through the files.

The parallax background image can be set in the main.css file - look for the #parallax div and replace the background image there. To change the scroll speed for the image, go to js/custom.js and you'll see the settings right at the top.

CSS Structure

Booom! was developed using LESS. The .less files are included in the template, although not used directly for the website. They allow you to customise the template via less if you wish to. To do so, you'll need to compile those files into the respective css files.

The index.html file calls one single css file, main.css. This file imports a few other css files, including the bootstrap css file. If you customise the css, you should place your css in main.less / main.css and leave the other files as is.

Upgrading to version 2.0

If you're not upgrading and will work straight with the version 2.0 files, you can skip this section.

Version 2.0 comes with some new features, and this implied some tiny tweaks to the HTML markup and Javascript custom file.

Before you upgrade, make sure you have a backup copy of your existing version 1.x files.


1. Logo Image Class

You will need to add the .logo-img class to the logo image, as follows:


  <!-- Logo goes here - replace the image with yours -->

  <a href="index.html" class="navbar-brand">
    <img class="logo-img" src="images/logo.png" alt="Booom! Logo" title="Booom! Logo">
  </a>

2. Gallery Items Markup

The gallery items will need to be wrapped in a <figure> tag and the anchor link within the figure needs a class of zoom.

You need to go from (version 1.x)

 
  <div class="col-lg-3 col-md-4 col-xs-6 gallery-item gallery-popup all kittens">
    <a href="http://www.placehold.it/640x480">
      <img src="http://www.placehold.it/640x480" alt="gallery-image" title="gallery-image" class="img-responsive" />
    </a>
  </div>

to (version 2.x)

 
  <div class="col-lg-3 col-md-4 col-xs-6 gallery-item gallery-popup all kittens">
    <figure>
      <a href="http://www.placehold.it/640x480" class="zoom">
        <img src="http://www.placehold.it/640x480" alt="gallery-image" title="gallery-image" class="img-responsive" />
      </a>
    </figure>
  </div>

Custom CSS File for Modifications

To make modifications easier in the future, i have added a custom stylesheet (LESS and CSS) named custom.css - this will allow to make modifications that do not get overwritten in future updates.

If you want to use this file, you need to call it in the <head> section of your current index.html file like so:

              
  <!-- The main CSS file -->
  <link href="css/main.css" rel="stylesheet">

  <!-- CSS file for your custom modifications -->
  <link href="css/custom.css" rel="stylesheet">

Portfolio Gallery

The portfolio gallery allows you to present your work in a fancy filtrable gallery (isotope plugin). You can either display your work in a popup image with a zoom effect, or take the user to a complete new project detail page.

To add a link to the project page, you need to add the following HTML to the gallery item, like it's demonstrated in a few of the items:


  <div class="item-hover">
    <a href="link-to-your-file.html">View Details</a>
  </div>

Javascript

In the /js/ folder, you'll find the custom.js which is the file where scripts are written. You can modify this file, but the other js files should, in most cases, be left alone.

With a bit of jQuery knowledge, in custom.js you'll be able to tweak the css animations on scroll, the twitter widget and the newsletter signup ajax form (see below).

Tooltips, Popovers & Modals

To display tooltips, popovers and modals, you'll need to use a class of respectively .tooltip-trigger, .popover-trigger, .modal-trigger. An example of each is displayed in the demo template, at the start of the info section.

AJAX Contact Form

To have the contact form working, you'll need to host your website on an APACHE server that can handle PHP files correctly. The main thing you need to do to get the form to work is to enter your desired e-mail address in the config.php file. This is where the notification of a new signup will be e-mailed. Please note that you'll need to create manually your mailing list with the incoming subscription e-mails.

The config.php file is in the root directory of the website and contains one line. Replace "your-email-address@here.com" with the appropriate address, that's all you need to do!

If you want to customise the error / success / notification messages, you can do so in /js/custom.js (for the success message) and in /assets/contact.php (for the error messages & e-mail confirmation).

Remember: this form will not work if server doesn't handle PHP files.

Twitter Widget

By default, the twitter widget is disabled because when not connected to a twitter account, it conflicts with other jquery plugins in the template. The span with an ID of #tweets-feed that is responsible to display the tweets has been commented out in index.html - Once you have set up the widget properly as explained below, you can uncomment that span.

To get your twitter feed working, you'll need to put your own twitter username in the settings. You can find the "Twitter Feed" section in /js/custom.js almost at the bottom of the file. Find the line "username" that has "envato" as a demo, and place your username id here instead. You can play around with other settings, such as display more than 1 tweet, change the "a few hours ago" text, etc.

You'll also need to create a Twitter App and enter your keys in the assets/twitter/lib/index.php file.

For full documentation on this twitter widget, please refer to https://github.com/StanScates/Tweet.js-Mod.
